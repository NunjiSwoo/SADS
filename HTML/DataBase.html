Banco de Dados

  • Página Inicial

    • Definição:

    Banco de dados é um sistema organizado de armazenamento de informações que permite o gerenciamento e a manipulação de dados, comumente em tabelas. Ele é projetado para armazenar grandes quantidades de informações de forma estruturada, acessível e eficiente.

    Estrutura de dados

    Uma estrutura de dados refere-se à maneira como os dados são organizados e armazenados internamente. Ela determina como os dados são representados e como as operações de leitura, escrita e manipulação são realizadas.

    • Dados Estruturadas

      • Elas tem um sistema fixo e consistente, como tabelas em um banco de dados relacional. Cada campo tem um tipo de dados específico e é armazenado em uma posição bem definida. Isso permite consultas e manipulações eficientes.

    • Semi estruturadas

      • Elas permitem flexibilidade na organização dos dados. Embora possuam alguma estrutura, ela não é rigidamente definida. Esses dados são geralmente representados em formatos como XML ou JSON.

    • Não Estruturadas

      • As estruturas de dados não estruturadas não possuem uma organização definida. São dados que não seguem um esquema ou formato específico. Eles são caracterizados pela ausência de estrutura predefinida e podem incluir texto livre, imagens, vídeos ou áudio.

    IDE

    O PostgreSQL é um Sistema de Gerenciamento de Banco de Dados Relacional (SGBDR) de código aberto amplamente utilizado e conhecido por sua confiabilidade, escalabilidade e recursos avançados, oferece também suporte completo ao padrão SQL. O PostgreSQL possui uma arquitetura flexível que permite a personalização e a criação de tipos de dados e funções personalizadas.

    Comandos

    Aqui esta uma lista sobre alguns comandos básicos utilizados na IDE postgreSQL, onde estes tratam basicamente do controle e manipulamento de uma ou mais tabelas.

    DQL:

    • SELECT

      • O comando SELECT é utilizado para recuperar dados de um banco de dados. Ele permite que você especifique as colunas que deseja selecionar, filtrar os registros com base em determinadas condições e ordenar os resultados.

    • FROM

      • O comando FROM é usado em conjunto com o comando SELECT para especificar a tabela ou tabelas das quais você deseja recuperar os dados. É aqui que você define a origem dos dados para a consulta.

    • WHERE

      • O comando WHERE é usado para filtrar os registros com base em condições especificadas. Com o WHERE, você pode definir critérios para selecionar apenas os registros que atendam a determinadas condições.

    DML:

    • INSERT

      • O comando INSERT é utilizado para inserir novos dados em uma tabela. Ele permite especificar os valores a serem inseridos nas colunas correspondentes.

    • UPDATE

      • O comando UPDATE é utilizado para modificar os dados existentes em uma tabela. Ele permite atualizar os valores das colunas especificadas em um ou mais registros com base em determinadas condições.

    • DELETE

      • O comando DELETE é utilizado para excluir registros de uma tabela. Ele permite remover um ou mais registros que atendam a determinadas condições.

    • MERGE

      • O comando MERGE é usado para combinar as operações de INSERT, UPDATE e DELETE em uma única instrução. Ele permite atualizar registros existentes se eles atenderem a determinadas condições ou inserir novos registros se não existirem.

    DDL

    • CREATE

      • Utilizado para criar uma nova tabela no banco de dados. Ao criar a tabela, você define os nomes das colunas, os tipos de dados que elas armazenarão e outras restrições, como chaves primárias ou estrangeiras.

    • ALTER

      • Este é utilizado para modificar a composição da estrutura de uma tabela, podendo adicionar, renomear ou remover colunas, alterar o tipo de dados de uma coluna, adicionar restrições, entre outras alterações.

    • DROP

      • Drop table é utilizado único e exclusivamente para excluir uma tabela e seus dados permanentemente de um banco de dados.

    DCL

    • GRANT

      • O comando GRANT é utilizado para conceder permissões e direitos de acesso a usuários ou funções em objetos de banco de dados. Ele permite especificar quais ações um usuário ou função pode realizar em determinados objetos dentro do banco de dados.

    • REVOKE

      • O comando REVOKE é usado para revogar ou remover as permissões ou direitos de acesso previamente concedidos a usuários ou funções. Ele permite remover as permissões específicas que foram concedidas anteriormente, negando o acesso aos objetos do banco de dados.

    • DENY

      • O comando DENY é utilizado para negar explicitamente as permissões a um usuário ou função em objetos de banco de dados. Ele difere do REVOKE, pois o REVOKE remove uma permissão que já foi concedida, enquanto o DENY nega explicitamente a permissão e impede que ela seja concedida novamente.

    DTL

    • COMMIT

      • Confirma a transação atual, tornando todas as alterações permanentes no banco de dados. As alterações feitas dentro da transação se tornam visíveis para outras transações.

    • ROLLBACK

      • Desfaz todas as alterações realizadas na transação atual e a descarta. O banco de dados retorna ao estado anterior à transação.

    • SAVEPOINT

      • Cria um ponto de salvamento dentro de uma transação para permitir o desfazer parcial das alterações.

    Dados e Chaves

    A IDE PostgreSQL pode comportar vários tipos de dados, dentre eles: números inteiros e decimais, binário, texto, booleano, etc.

    • Chave Primária

      • É basicamente um ou mais campo dentro da tabela que se torna um identificador único da tabela, tornando assim os elementos de uma tabela identificáveis se a chance de uma busca por exemplo apontar para duas informações com dados duplicados.

    • Chave Estrangeira

      • Já a chave estrangeira é praticamente uma chave primaria que foi inserida dentro de outra tabela, isto ocorre a fim de exibir uma relação que esta tabela tem com a da chave primária, de forma que toda tabela deve ter uma chave primária e uma estrangeira é opcional de acordo com a situação.

    Relacionamentos

    • Um para Um

      • Um para um é um relacionamento onde um registro/linha, se conecta somente com outra linha em outra tabela

    • Um para Muitos

      • Um para muitos é um relacionamento similar ao um para um, exceto que ele relaciona um registro com n outros.

    • Muitos para Muitos

      • Muitos para Muitos seria oposto ao um para um, de forma que ele relaciona n registro com n outros.
