Banco de Dados

  • Página Inicial

    • Definição:

    Banco de dados é um sistema organizado de armazenamento de informações que permite o gerenciamento e a manipulação de dados, comumente em tabelas. Ele é projetado para armazenar grandes quantidades de informações de forma estruturada, acessível e eficiente.

    Estrutura de dados

    Uma estrutura de dados refere-se à maneira como os dados são organizados e armazenados internamente. Ela determina como os dados são representados e como as operações de leitura, escrita e manipulação são realizadas.

    • Dados Estruturadas

      • Elas tem um sistema fixo e consistente, como tabelas em um banco de dados relacional. Cada campo tem um tipo de dados específico e é armazenado em uma posição bem definida. Isso permite consultas e manipulações eficientes.

    • Semi estruturadas

      • Elas permitem flexibilidade na organização dos dados. Embora possuam alguma estrutura, ela não é rigidamente definida. Esses dados são geralmente representados em formatos como XML ou JSON.

    • Não Estruturadas

      • As estruturas de dados não estruturadas não possuem uma organização definida. São dados que não seguem um esquema ou formato específico. Eles são caracterizados pela ausência de estrutura predefinida e podem incluir texto livre, imagens, vídeos ou áudio.

    IDE

    O PostgreSQL é um Sistema de Gerenciamento de Banco de Dados Relacional (SGBDR) de código aberto amplamente utilizado e conhecido por sua confiabilidade, escalabilidade e recursos avançados, oferece também suporte completo ao padrão SQL. O PostgreSQL possui uma arquitetura flexível que permite a personalização e a criação de tipos de dados e funções personalizadas.

    Comandos

    Aqui esta uma lista sobre alguns comandos básicos utilizados na IDE postgreSQL, onde estes tratam basicamente do controle e manipulamento de uma ou mais tabelas.

    • CREATE TABLE

      • Utilizado para criar uma nova tabela no banco de dados. Ao criar a tabela, você define os nomes das colunas, os tipos de dados que elas armazenarão e outras restrições, como chaves primárias ou estrangeiras.

    • DELETE

      • DELETE tem como função deletar algo em um banco de dados, de forma que sempre seja visado um cuidado ao utilizar este comando, uma vez que pode excluir toda base de dados quando usado ad maneira incorreta.

    • ALTER TABLE

      • Este é utilizado para modificar a composição da estrutura de uma tabela, podendo adicionar, renomear ou remover colunas, alterar o tipo de dados de uma coluna, adicionar restrições, entre outras alterações.

    • DROP TABLE

      • Drop table é utilizado unico e exclusivamente para excluir uma tabela e seus dados premanentemente de um banco de dados.

    Dados e Chaves

    A IDE PostgreSQL pode comportar vários tipos de dados, dentre eles: numeros inteiros e decimais, binário, texto, booleano, etc.

    • Chave Primária

      • É basicamente um ou mais campo dentro da tabela que se torna um identificador unico da tabela, tornando assim os elementos de uma tabela identificaveis se a chance de uma busca por exemplo apontar para duas informações com dados duplicados.

    • Chave Extrangeira

      • Já a chave estrangeira é praticamente uma chave prímaria que foi inserida dentro de outra tabela, isto ocorre a fim de exibir uma relação que esta tabela tem com a da chave primária, de forma que toda tabela deve ter uma chave primária e uma estrangeira é opcional de acordo com a situação.

    Relacionamentos

    • Um para Um

      • Um para um é um relacionamento onde um registro(uma linha), se conecta somente com outra linha em outra tabela

    • Um para Muitos

    • Muitos para Muitos
