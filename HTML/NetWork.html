    • Definição

    Refere-se a um conjunto de dispositivos interconectados, como destinatários, servidores, roteadores e switches, que se comunicam entre si para compartilhar recursos, como arquivos, impressoras, aplicativos e conexões com a Internet, muito disto através do modelo TCP/IP que é um conjunto de protocolos de comunicação que permite a interconexão de redes de destinatários.

    Experiência:

    Na matéria de redes compreendemos o conceito e como a internet e outras transmições e dado funcionam, de forma geral vimos sobre o modelo OSI e TCP/IP, quais nos explicam as camadas pelas quais a informação deve vaijar para que possa ser acessada por outro dispositivo, isto é algo que sempre que revisito ou me lembro é como se eu interpretasse meu arredor de uma maneira diferente, rever conceitos tão mundanos como estes e perceber quão complexo são é intrigante.

    História da Internet

    A Internet foi criada nas décadas de 1960 e 1970 quando diferentes redes de computadores foram conectadas pela primeira vez, o projeto inicial foi nomeado como: ARPANET, foi desenvolvido nos Estados Unidos pelo Departamento de Defesa para criar uma rede de comunicação segura mesmo durante possíveis ataques nucleares, com o passar do tempo, a Internet cresceu e se tornou uma rede global, conectando computadores em todo o mundo.

    Modelo OSI

    Camada Aplicativos:

    Esta camada é responsável pela interação direta com os aplicativos e usuários finais. Ela fornece serviços de rede para aplicativos, como correio eletrônico, transferência de arquivos e acesso a bancos de dados.

    Camada Apresentação:

    Esta camada trata da representação dos dados, garantindo que os dispositivos de origem e destino possam interpretar corretamente as informações transmitidas. Ela lida com a criptografia, compressão e formatação dos dados.

    Camada Sessão:

    Esta camada estabelece, mantém e encerra as conexões entre os aplicativos em diferentes dispositivos. Ela gerencia as sessões de comunicação, permitindo que os aplicativos troquem dados de maneira organizada.

    Camada Transporte:

    Esta camada é responsável por dividir os dados em pacotes e garantir que eles sejam entregues corretamente ao destino. Ela controla a confiabilidade da transmissão, a detecção de erros e a retransmissão, se necessário.

    Camada Rede:

    A função principal dessa camada é controlar o roteamento dos dados através de uma rede. Ela determina o melhor caminho para a transmissão dos dados e lida com o endereçamento lógico dos dispositivos na rede.

    Camada Enlace:

    Esta camada fornece um meio confiável de transferência de dados entre dois dispositivos diretamente conectados. Ela cuida do endereçamento físico, do controle de fluxo e da detecção de erros.

    Camada Física:

    Esta camada lida com a transmissão física dos dados. Ela define as especificações elétricas, mecânicas e de hardware para a transmissão dos dados através dos meios físicos, como cabos e sinais.

    Topologia em redes

    1. Topologia em Barramento:

      2. Nesta topologia, todos os dispositivos são conectados a um único cabo compartilhado. Os dados são transmitidos pelo cabo e todos os dispositivos recebem esses dados, mas apenas o destinatário pretendido os processa.

    2. Topologia em Estrela

      3. Nesta topologia, todos os dispositivos são conectados a um ponto central, geralmente um switch ou concentrador. Os dispositivos se comunicam enviando dados para o ponto central, que os direciona para o destinatário correto.

    3. Topologia Mista

      4. Nessa topologia, é utilizada uma combinação de diferentes topologias, como estrela, barramento , anel, etc. Isso ocorre quando diferentes partes da rede são conectadas usando topologias diferentes, dependendo dos requisitos específicos de cada área.

    4. Topologia em Anel

      5. Nesta topologia, os dispositivos são conectados em uma forma de anel, onde cada dispositivo está conectado a outros dois dispositivos. Os dados são transmitidos em uma direção ao redor do anel, passando por cada dispositivo até chegar ao destinatário desejado.

    5. Topologia em Anel Duplo:

      6. Nesta topologia, os dispositivos são conectados em dois anéis separados, geralmente em direções opostas. Isso oferece redundância e aumenta a disponibilidade da rede, pois se um anel falhar, a comunicação pode ser redirecionada para o outro anel.

    6. Topologia em Árvore:

      7. Nessa topologia, os dispositivos são organizados hierarquicamente em uma estrutura de árvore. Um dispositivo central, como um switch ou roteador, é conectado a dispositivos secundários, que por sua vez estão conectados a dispositivos adicionais. Isso permite a criação de redes maiores e mais organizadas.

    7. Topologia em Malha:

      8. Nesta topologia, todos os dispositivos são conectados diretamente a todos os outros dispositivos da rede. Isso cria múltiplos caminhos de comunicação, o que aumenta a redundância e a confiabilidade da rede.

    8. Topologia Totalmente Conectada:

      9. Nessa topologia, todos os dispositivos estão diretamente conectados a todos os outros dispositivos da rede. Isso cria uma rede altamente redundante e confiável, onde cada dispositivo tem uma conexão direta com todos os outros.

    Transmissão de dados

    • Síncrona:

      • Neste tipo de transmissão, os dados são enviados em um ritmo constante, seguindo um relógio compartilhado entre o transmissor e o receptor. Os dados são transmitidos em blocos, chamados de quadros, e são sincronizados com o ritmo do relógio. Isso significa que o transmissor e o receptor estão alinhados no tempo durante toda a transmissão.

    • Assíncrona:

      • Neste tipo de transmissão, os dados são enviados de forma independente, sem uma conexão de sincronização rígida entre o transmissor e o receptor. Cada caractere ou bloco de dados é enviado individualmente, com informações adicionais para identificar o início e o fim de cada transmissão. Isso permite que os dispositivos enviem dados em seu próprio ritmo, sem depender de um relógio compartilhado.

    • Série:

      • Na transmissão de dados em série, os bits de dados são enviados um após o outro, em uma única linha de comunicação. Cada bit é transmitido sequencialmente, geralmente em uma sequência de 0s e 1s.

    • Paralela:

      • Na transmissão de dados em paralelo, vários bits de dados são transmitidos simultaneamente, utilizando varias linhas de comunicação. Cada bit é enviado em uma linha separada, permitindo que vários bits sejam transmitidos ao mesmo tempo.

    • Simplex:

      • Na transmissão simplex, a comunicação ocorre em apenas uma direção. Um dispositivo transmite dados para outro dispositivo, mas não há uma transmissão bidirecional.

    • Half-Duplex:

      • Na transmissão half-duplex, a comunicação ocorre em ambas as direções, mas não simultaneamente. Os dispositivos podem transmitir e receber dados, mas em momentos diferentes.

    • Full-Duplex:

      • Na transmissão full-duplex, a comunicação ocorre em ambas as direções simultaneamente. Os dispositivos podem transmitir e receber dados ao mesmo tempo.

    Codificação

    • NRZ:

      • Na codificação NRZ, cada bit de dados é representado por um nível de tensão constante ao longo de um determinado período de tempo. Um bit "1" é representado por uma tensão alta ou positiva, enquanto um bit "0" é representado por uma tensão baixa ou negativa.

    • Manchester:

      • Na codificação Manchester, cada bit de dados é dividido em duas fases dentro de um determinado período de tempo. Um bit "1" é representado por uma transição da tensão do nível alto para o nível baixo no meio do período e, em seguida, uma transição de volta ao nível alto. Um bit "0" é representado por uma transição da tensão do nível baixo para o nível alto no meio do período e, em seguida, uma transição de volta ao nível baixo.

    Transmissão Diferencial

    Nela dois sinais são usados: um sinal positivo e um sinal negativo. A diferença de tensão entre esses dois sinais representa os dados a serem transmitidos. Se o sinal positivo estiver em um nível mais alto do que o sinal negativo, isso representa um "1", e se o sinal positivo estiver em um nível mais baixo do que o sinal negativo, isso representa um "0".

    Verificação de dados

    Durante a transmissão ou armazenamento de dados, podem ocorrer erros devido a ruídos, interferências ou problemas técnicos. A verificação de dados ajuda a detectar esses erros para garantir a integridade dos dados.

    Paridade de dados

    Na paridade de dados, um bit adicional, chamado de bit de paridade, é adicionado aos dados. Esse bit é calculado com base na quantidade de bits "1" presentes nos dados, no final ao receber os dados, o destinatário recalcula o bit de paridade e o compara com o bit recebido, caso haja diferença entre eles, isso indica que ocorreu um erro de transmissão.

    • Par

      • O bit de paridade é definido como "1" se o número de bits "1" nos dados for ímpar. Caso contrário, o bit de paridade é definido como "0".

    • Ímpar

      • Já quando é ímpar, o bit de paridade é definido como "1" se o número de bits "1" for par. Caso contrário, o bit de paridade é definido como "0".

    CRC

    O CRC é um método usado para verificar se os dados estão corretos. Ele adiciona um código específico aos dados que é comparado quando os dados são recebidos novamente. Se os códigos coincidirem quer dizer que os dados estão bons.

    Tipos de redes:

    • WAN

      • É uma grande rede que conecta computadores e redes em diferentes lugares, como cidades, países ou até mesmo em todo o mundo.

    • MAN

      • É uma rede que cobre uma área metropolitana, como uma cidade, e conecta vários locais próximos.

    • CAN

      • É uma rede que abrange um campus, como uma universidade, e conecta diferentes prédios e departamentos.

    • LAN

      • É uma rede local que cobre um espaço menor, como uma casa, escritório ou escola, e permite que os dispositivos se comuniquem entre si.

    • PAN

      • É uma rede pessoal que conecta dispositivos pessoais, como smartphones, tablets e laptops, geralmente em curta distância, como dentro de uma sala.

    Endereço MAC

    Endereço MAC é um tipo de identificação aplicada em todos os tipos de dispositivos conectáveis à internet, de forma que sempre quando conectado em uma LAN seja possível quantificar e saber quem usa a mesma através dele.

    Quadro Ethernet

    Um quadro Ethernet é uma unidade de dados de protocolo essencial em uma rede Ethernet, de forma que é o responsável por armazenar os dados transferidos em uma LAN, possuindo informações sobre os pacotes e MACs de origem e final.

    Endereço IPv4:

    O endereço IPv4 é um sistema de endereçamento utilizado na Internet, ele é composto por um conjunto de quatro números separados por pontos, cada número variando de 0 a 255. Por exemplo, um exemplo pode ser representado por 192.168.0.1.

    NAT:

    É uma técnica utilizada em redes de computadores para permitir que vários dispositivos compartilhem um único endereço IP público, de forma que economize IPs.

    DHCP

    É um protocolo usado para atribuir automaticamente endereços IP e outras configurações de rede, em vez de configurar manualmente cada dispositivo com as informações de rede.

    Endereços reservados:

    Eles são endereços IP reservadas para uso interno em redes locais, não são roteáveis pela Internet pública e são usados principalmente em redes domésticas e empresariais.

    Endereço IPv6

    O endereço IPv4 possui um limite, pois utiliza endereços com 32 bits, que uma hora ou outra irão acabar, para isso foi criado o endereço IPv6, qual possui 128 bits de endereço, este também possui Multicast, anycast e um DHCPv6 próprio para si.

    Unicast Link-Local:

    Este é um dispositivo que é utilizado para comunicações em uma mesma rede física, não sendo roteável parar internet.

    Unicast Unique-Local:

    Por outro lado este é utilizado para comunicação em redes privadas, sendo semelhante aos endereços IPv4 reservados e também não é roteável para internet.

    IANA

    Ela é a Autoridade para Atribuição de Números da Internet (AANI), em português, sendo responsável pela coordenação e alocação de recursos críticos da Internet, trabalhando com as cinco Registros Regionais da Internet (RIRs) para distribuir e gerenciar esses recursos em nível global.

    LACNIC

    Este é um dos cinco Registros Regionais da Internet (RIRs) em todo o mundo, é responsável pela distribuição e gerenciamento de recursos de endereçamento IP, números de sistemas autônomos, e outros recursos relacionados à Internet para a região da América Latina e Caribe.

    DNS

    é um sistema que permite a tradução de nomes de domínio em endereços IP, ele é quem nos permite usar nomes de domínio amigáveis, como: www.google.com.

    TDL

    É a parte mais alta e final de um nome de domínio na Internet, sendo a porção que aparece à direita do último ponto no endereço do site, eles são usados para categorizar e identificar tipos de sites e sua afiliação geográfica ou funcional.

    Protocolos

    • TCP

      • É um protocolo que estabelece uma conexão entre um remetente e um destinatário, dividindo os dados em pacotes e garantindo que eles cheguem na ordem correta.

    • UDP

      • É um protocolo sem tanta confiança, pois envia pacotes de dados sem garantir sua entrega ou ordem, porém ele é usado quando o importante é a velocidade.

    • SCTP

      • É um protocolo que combina conceitos dos TCP e UDP, ele é confiável como o TCP e suporta transmissões paralelas de pacotes.

    • DCCP

      • É um protocolo permite a entrega não confiável e rápida de pacotes junto de mecanismos para controlar a congestão de rede.

    QoS

    É um conjunto de mecanismos e técnicas usados para gerenciar a qualidade da transmissão de dados em redes de computadores, gestando o tráfego de acordo com requisitos específicos, como largura de banda, latência, confiabilidade e segurança, ajudando a garantir que determinados tipos de dados recebam tratamento preferencial.

    ToS

    É um campo no cabeçalho IP que identifica o tipo de serviço ou nível de prioridade atribuído a um pacote de dados, sendo usado para categorizar o tráfego de rede com base em requisitos de qualidade e tratamento diferenciado.

    DSCP

    É um valor atribuído no campo ToS do cabeçalho IP que especifica o tratamento preferencial e o encaminhamento de um pacote de dados em uma rede, permitindo que os pacotes sejam classificados em diferentes categorias ou classes de serviço com base em suas necessidades de qualidade.

    ECN

    É um mecanismo usado em redes para evitar o congestionamento e melhorar o desempenho, em vez de esperar que os pacotes sejam descartados quando a rede está congestionada, o ECN permite que os dispositivos de rede comuniquem explicitamente o congestionamento.

    Aplicações

    São programas ou serviços que utilizam a infraestrutura de rede para realizar tarefas específicas. Essas aplicações permitem que dispositivos se comuniquem, compartilhem informações e executem diversas atividades.

    HTTP

    É um protocolo usado para enviar e receber informações na World Wide Web, permitindo que navegadores se conectem a servidores web, solicitem páginas da web e exibam conteúdo.

    SSH

    É um protocolo seguro de comunicação usado para acessar e gerenciar remotamente servidores e dispositivos, fornecendo autenticação e criptografia para proteger a conexão e permite que os usuários controlem servidores

    SMTP

    É um protocolo usado para enviar e entregar e-mails entre servidores de e-mail.

    NTP

    É um protocolo usado para sincronizar relógios de dispositivos em redes de computadores, permitindo que os dispositivos obtenham e mantenham a mesma hora e data precisa.

    Componentes

    Switch

    Switches: Os switches são dispositivos de rede que conectam vários dispositivos, como computadores, impressoras e servidores, em uma LAN.

    Hub

    Os hubs são dispositivos mais simples que também conectam vários dispositivos em uma rede local, Entretanto, eles não possuem a capacidade de direcionar o tráfego de forma inteligente e analisar endereços MAC.

    Roteador

    É responsável por encaminhar o tráfego entre redes, de forma que ele determina o caminho mais eficiente para os pacotes de dados viajarem.

    Servidores

    Estes são computadores poderosos que fornecem serviços e recursos para outros dispositivos em uma rede, eles armazenam e compartilham arquivos, hospedam sites e executam aplicativos.

    Nuvem

    Ela é basicamente um conjunto de serviços e recursos disponibilizados através da internet, em vez de armazenar dados ou executar aplicativos localmente a computação em nuvem permite que os usuários acessem e utilizem recursos remotos.

    Fire Wall

    É um componente de segurança que protege a rede controlando o tráfego de entrada e saída, monitorando e filtrando os pacotes de dados com base em regras de segurança já predefinidas.
